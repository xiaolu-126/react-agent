## 一、流程介绍

### 1.1 整体架构

本项目采用 **ReAct（Reasoning + Acting）架构**，基于 LangGraph 状态图（StateGraph）编排 Agent 的推理与执行流程。系统由以下核心模块组成：

```
┌─────────────────────────────────────────────────────────────┐
│                    用户接口层                                  │
│   ┌────────────┐  ┌────────────┐  ┌──────────────────────┐   │
│   │ CLI 命令行  │  │ FastAPI    │  │ SSE 流式接口          │   │
│   └─────┬──────┘  └─────┬──────┘  └─────────┬────────────┘   │
├─────────┼───────────────┼───────────────────┼────────────────┤
│         ▼               ▼                   ▼                 │
│                    ReAct Agent (LangGraph)                    │
│   ┌────────────────────────────────────────────────────┐     │
│   │                    状态图循环                         │     │
│   │  ┌──────┐    ┌──────┐    ┌──────────┐    ┌──────┐   │     │
│   │  │Agent │───▶│Action│───▶│Observa- │───▶│Agent │───▶  │     │
│   │  │ 节点  │    │ 节点  │    │tion节点  │    │ 节点  │  ...  │     │
│   │  └──┬───┘    └──┬───┘    └────┬─────┘    └──┬───┘   │     │
│   │     │            │             │             │       │     │
│   │     ▼            ▼             ▼             ▼       │     │
│   │  思考/推理     调用工具     观察结果     生成最终输出   │     │
│   └────────────────────────────────────────────────────┘     │
├─────────────────────────────────────────────────────────────┤
│                     模型与工具层                               │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│   │ 模型管理器 │  │ 知识库    │  │ 网络搜索  │  │ 对话记忆  │   │
│   │(ModelMgr)│  │(ChromaDB)│  │(Serper)  │  │(Memory)  │   │

└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心流程详解

系统采用 **ReAct（Reasoning + Acting）** 循环推理机制，完整流程如下：

#### 步骤 1：状态初始化
- 接收用户输入和主播名称等参数
- 从对话记忆管理器加载历史消息
- 构建初始状态 `AgentState`

#### 步骤 2：Agent 推理节点
- 加载系统提示词（System Prompt）
- 结合对话历史（过滤后）和用户输入
- 调用 LLM 进行推理决策
- 输出可能是**最终回答**或**工具调用请求**

#### 步骤 3：条件分支判断
```
┌─────────────────────────────┐
│        Agent 输出           │
├─────────────────────────────┤
│ 是否包含 tool_calls?        │
│   ├── 是 → 跳转到 Action 节点 │
│   └── 否 → 跳转到 END (输出)  │
└─────────────────────────────┘
```

#### 步骤 4：工具执行（Action 节点）
- 支持的工具：
  - **知识库搜索**：基于 ChromaDB 的向量相似度搜索
  - **网络搜索**：通过 Serper API 获取实时信息
- 执行结果以 ToolMessage 形式返回

#### 步骤 5：观察记录（Observation 节点）
- 收集本轮所有工具执行结果
- 记录到状态中的 observations 列表
- 为下一轮 Agent 推理提供上下文

#### 步骤 6：循环迭代
- 返回 Agent 节点进行下一轮推理
- 直到 Agent 不再调用工具，直接输出最终结果
- 递归限制：最多 25 次迭代

#### 步骤 7：输出后处理
- 多策略去重算法（5 种策略）
- 通过对话记忆管理器存储本轮交互
- 返回最终结果

### 1.3 API 接口设计

| 接口路径 | 方法 | 功能 | 流式 |
|---------|------|------|------|
| `/chat` | POST | 非流式聊天 | 否 |
| `/chat/stream` | POST | 流式聊天 | 是（SSE） |
| `/generate` | POST | 生成主播推荐 | 否 |
| `/generate/stream` | POST | 流式生成推荐 | 是（SSE） |
| `/models` | GET | 获取模型列表 | - |
| `/models/switch` | POST | 切换模型 | - |
| `/knowledge/upload` | POST | 上传文档到知识库 | - |
| `/knowledge/search` | POST | 搜索知识库 | - |
| `/system-prompts` | GET/POST | 管理提示词 | - |
| `/memory/clear` | POST | 清空对话记忆 | - |

### 1.4 数据流示例

```
用户: "推荐主播 8歌"
  │
  ▼
Agent.run(streamer_name="8歌")
  │
  ├── 加载系统提示词（主播推荐专家）
  ├── 构造状态: {streamer_name: "8歌", input: "请推荐主播"}
  │
  ├── 第一轮 Agent 推理
  │   ├── LLM 决定：搜索知识库获取"8歌"信息
  │   └── 输出工具调用 → 执行搜索
  │
  ├── 第二轮 Agent 推理
  │   ├── LLM 结合搜索结果生成推荐
  │   └── 输出最终推荐理由
  │
  └── 去重处理 → 返回最终结果
      🎙️ 8歌
      风格：...
      一句话推荐：...
```

本项目集成了**两种搜索能力**：

1. **知识库搜索（Knowledge Base Search）**：基于 ChromaDB 的向量化语义搜索，用于检索已上传文档
2. **网络搜索（Web Search）**：通过 Serper API 获取实时网络信息

## 五、总结

本项目基于 LangGraph 框架，采用 ReAct 架构构建了一个**低成本、高灵活性、完全开源**的智能 Agent 系统。与市面上的商业 Agent 平台相比，核心优势在于：

1. **零平台费用**：仅需承担模型 API 调用成本，比商业平台节省 50%~90% 的费用
2. **多模型自由切换**：支持 OpenAI、Anthropic、通义千问、文心一言、DeepSeek 等主流模型
3. **数据安全可控**：全链路本地部署，知识库数据不离开自有环境
4. **灵活自定义**：系统提示词、工具、输出格式均可按需定制
5. **高性价比**：搭配 DeepSeek V3 等国产模型，百万 Token 输出成本低至 ¥8
